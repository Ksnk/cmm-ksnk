
Для установки ловушек используется скрипт "Перебор По Списку(2).смм"

Для того, чтобы настроить скрипт для конкретной ситуации, нужно определить переменные 
mod1,mod2... mod7. Значниями этих переменых должны быть номера модулей, для которых проводится операция.
Каждая переменная должна содержать не более 4 номеров. к примеру, для варианта:
  mod1 := $70717273 ;mod2 := $74757677;mod3 := $04 ;
будут последовательно перебираться модули с номерами 
$73,$72,$71,$70,$77,$76,$75,$74,$04
Таким обазом, всего может быть не более, чем 7*4=28 модулей.

Для того, чтобы: 
- поставить ловушки, нужно после запуска скрипта указать ему "1". 
- проверить рестарты - '2'. при этом будет выводится список модулей, после каждого номера символ '.' - не было ничего, '-' - сработала динамическая довушка 'x' - произошел рестарт модуля.
- проверять рестарты циклически - '$12'
- остальные значения для задачи сбора информаци о рестарте не используются.

Получение результата - скрипт "Получить результат.смм".
после того как ловушки сработают, нужно запустить скрипт "ПОлучить результат.смм" и указать ему нужный номер модуля. После окончания исполнения скрипта, файл "получить результат.log" нужно переслать разработчикам для анализа.
